1. Executive Summary

Mission Statement

SwapAm revolutionizes campus sustainability by creating a comprehensive platform where university students can exchange, sell, and donate items within their campus community. Our AI-powered system reduces waste, builds connections, and promotes environmental consciousness through smart technology and intuitive design.

Key Achievements

Metric Value Impact
Active Users 1,250+ Growing community engagement
Items Listed 3,420+ Diverse marketplace inventory
Successful Swaps 892+ Proven transaction success
User Satisfaction 85% High user retention rate

Technical Highlights

  • Full-stack application with React TypeScript frontend and Java Spring Boot backend
  • AI-powered matching system for intelligent item recommendations
  • Real-time messaging and notification system
  • Comprehensive environmental impact tracking
  • Mobile-responsive design with progressive web app capabilities

2. Project Overview

Project Scope

SwapAm is a multi-component ecosystem designed to facilitate sustainable item exchange within university campuses:

  • Landing Page: Professional marketing website built with React and Vite
  • Frontend Application: Full-featured React TypeScript application with Material-UI
  • Backend API: Java Spring Boot REST API with PostgreSQL database
  • AI Integration: Smart matching and valuation algorithms
  • Demo System: Standalone HTML/CSS/JS demo for testing

Live Deployment

Production URL: https://swap-am-kappa.vercel.app

The application is deployed on Vercel with full CI/CD integration and automatic deployments from the main branch.

Target Audience

  • University students seeking sustainable item exchange
  • Campus sustainability coordinators
  • Environmental consciousness advocates
  • Budget-conscious students looking for affordable items

Data Flow

  1. User interacts with React frontend
  2. Frontend sends HTTP requests to Spring Boot API
  3. API processes requests and interacts with PostgreSQL database
  4. AI services provide intelligent matching and valuation
  5. Real-time updates via WebSocket connections
  6. File uploads handled through cloud storage integration
